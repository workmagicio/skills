## Canonical interpretation language

Reusable patterns. Match the user's situation to a row → use the corresponding business-language explanation as a starting point.

<table><colgroup><col/><col/></colgroup><thead><tr><th vertical-align="top"><b>Pattern</b></th><th vertical-align="top"><b>Business-language explanation</b></th></tr></thead><tbody><tr><td vertical-align="top">MBO recommends increasing channel X</td><td vertical-align="top">"Channel X's saturation curve is still steep at current spend — there's headroom for additional dollars to keep producing meaningful incremental returns. The recommendation moves you toward equilibrium where every channel's next dollar produces a similar incremental outcome."</td></tr><tr><td vertical-align="top">MBO recommends cutting channel X</td><td vertical-align="top">"Channel X is near saturation — its curve is flat at current spend, meaning each additional dollar adds little. The model reallocates that spend to channels still on the steep part of their curves."</td></tr><tr><td vertical-align="top">High average ROAS but low marginal ROAS</td><td vertical-align="top">"Average ROAS blends efficient early spend with saturated late spend. Channel X's average looks healthy, but marginal ROAS — return on the next dollar — has collapsed at current spend. That's why MBO suggests a cut."</td></tr><tr><td vertical-align="top">Baseline (organic) sales large vs paid media sales</td><td vertical-align="top">"A big share of your sales is organic / recurring demand independent of paid media. MBO doesn't optimize that baseline — it only reallocates the paid portion. Reading the recommendation requires looking at the paid component separately."</td></tr><tr><td vertical-align="top"><b>Total projected sales LOWER than reference, but goal is "maximize sales"</b></td><td vertical-align="top"><b>Almost always a baseline issue, not a recommendation issue.</b> Decompose:<ul><li><b>Paid component</b>: up or down? If up (and ROAS / marginal ROAS improved) → reallocation IS working — that's what MBO controls</li><li><b>Baseline component</b>: if down → that's where the total drop comes from. MBO doesn't optimize baseline (organic + seasonality projection)</li><li><b>Frame</b>: "Paid media sales are up $X (+Y%) and ROAS improved A → B — reallocation is doing its job. Total is down $Z because baseline projection is Q% lower (seasonal / organic decline). MBO can't optimize baseline, so this drop is outside the recommendation's control."</li><li>If backtesting confidence is low → add: "Treat absolute projection as directional, not literal"</li></ul><br/><b>Never say</b> "reference baseline reflects channel mix" or "optimizer finds more efficient distribution" without first checking baseline — generic and misleading.</td></tr><tr><td vertical-align="top"><b>User has manually locked several tactics; total projected sales is lower than reference</b></td><td vertical-align="top"><b>Don't blame the user's own locks for the drop.</b> Locks are intent, not flaws.<ul><li>Treat locks as fixed boundary conditions; don't list them as the cause</li><li>Run the baseline / paid decomposition first — the drop is usually baseline, not the locks</li><li>Within unlocked tactics, explain what MBO did do</li><li>Only mention locks if user explicitly asks "why didn't the model optimize tactic X" — name them neutrally as scope, not as problem</li></ul><br/><b>Never say</b>: "the model is solving best plan given these locks, not best plan overall, that's why sales drop"</td></tr><tr><td vertical-align="top">Recommendations differ from a run a few days ago (same inputs)</td><td vertical-align="top">"Saturation curves refresh daily. Even a few days can shift a channel's curve, particularly near saturation inflection. New lift tests also recalibrate affected channels. Expected behavior."</td></tr><tr><td vertical-align="top">MBO vs attribution dashboard for same period</td><td vertical-align="top">"MBO shows model-estimated outcomes at reference spend; dashboard shows realized actuals using last-click / iDDA. Two methodologies, expected to differ. Dashboard for performance tracking, MBO for allocation."</td></tr><tr><td vertical-align="top">iDDA strong recently, MBO forecast lower</td><td vertical-align="top">"iDDA shows realized iROAS — fluctuates with promos, creative, seasonality. MBO fits a stable curve to smooth those spikes, predicting the sustainable future. Gap is logical, not a discrepancy."</td></tr><tr><td vertical-align="top">Backtesting accuracy &lt; 70%</td><td vertical-align="top">"Backtesting accuracy is [X]%, below recommended 70%. Use recommendations as directional guidance. Your CS team can review what's driving lower accuracy."</td></tr><tr><td vertical-align="top">Lift test iROAS vs MBO marginal ROAS differ</td><td vertical-align="top">"Lift test measures incremental ROAS (causal contribution at tested spend). MBO marginal ROAS measures return on the next dollar at recommended spend. Complementary, not competing — lift test is used to calibrate MBO."</td></tr><tr><td vertical-align="top">Reference period spend = 0 for a channel</td><td vertical-align="top">"Channel X had zero spend in the reference window — MBO can't model from zero baseline, so it's excluded. To include, run a scenario with a reference window where you actually spent on it."</td></tr><tr><td vertical-align="top">Insufficient data for saturation curve</td><td vertical-align="top">"Channel Y doesn't have enough historical data for a reliable curve, so MBO held budget at baseline (no change). Once more history accumulates, future scenarios can optimize it."</td></tr></tbody></table>
